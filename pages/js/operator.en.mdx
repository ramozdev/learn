# Operator

## Destructuring

### Arrays

#### Without destructuring

```js copy
const colors = ['blue', 'yellow', 'black']

// without destructuring
const firstElement = colors[0]
const secondElement = colors[1]
const thirdElement = colors[2]

console.log(firstElement, secondElement, thirdElement) // 'blue yellow black'
console.log(`The sky is ${firstElement}`) // 'The sky is blue'
```

#### With destructuring

```js copy
const colors = ['blue', 'yellow', 'black']

const [firstColor, secondColor, thirdColor] = colors

console.log(firstColor, secondColor, thirdColor) // 'blue yellow black'
console.log(`The sun is ${secondColor}`) // 'The sun is yellow'
```

### Objects

#### Without destructuring

```js copy
const person = { firstName: 'Alex', weight: 140, age: 23 }

// without destructuring
const firstName = person.firstName
const weight = person.weight
const age = person.age

console.log(firstName, weight, age) // 'Alex 140 23'
console.log(`${firstName} weighs ${weight} pounds and is ${age} years old.`)
// 'Alex weighs 140 pounds and is 23 years old.'
```

#### With destructuring

```js copy
const post = { title: 'Drink More Water', author: 'Juan Pérez', views: 539 }

const { title, author, views } = post

console.log(title) // 'Drink More Water'
console.log(`${author} wrote "${title}". This post has ${views} views.`)
// 'Juan Pérez wrote "Drink More Water". This post has 539 views.'
```

### Rest property (Spead syntax, ...rest)

You can destructure a subset of the properties and create a new object with the rest of the properties.

#### Objects

```js copy
const comment = { body: 'I love it!', likes: 20, by: 'Peter' }

const { body, ...rest } = comment

console.log(body) // 'I love it!'
console.log(rest) // { likes: 20, by: 'Peter' }
```

#### Arrays

```js copy
const colors = ['red', 'green', 'blue']

const [firstColor, ...rest] = colors

console.log(firstColor) // red
console.log(rest) // ['green', 'blue']
```

You can rename a property you are destructuring.

```js copy
const comment = { body: 'I love it!', likes: 20, by: 'Peter' }

const { by: author, ...rest } = comment

console.log(author) // 'Peter'
console.log(rest) // { body: 'I love it!', likes: 20 }
```

You set a default value to replace `undefined{:js}` values.

```js copy
const profile = {
  username: 'justatable11',
  followers: 1000,
  following: 500,
  bio: undefined,
  coordinates: undefined
}

const {
  bio = `I'm keeping quiet.`,
  coordinates: location = 'I keep my location private.',
  ...props
} = profile

console.log(bio) // 'I'm keeping quiet.'
console.log(location) // 'I keep my location private.'
console.log(props)
// { username: 'justatable11', followers: 1000, following: 500 }

const fruits = [undefined, 'banana', 'orange', 'pear']

const [first = 'APPLE', second, ...others] = fruits

console.log(first) // 'APPLE'
console.log(second) // 'banana'
console.log(others) // [ 'orange', 'pear' ]
```

## Comparison operators

### Equal (==)

Returns true if the operands are equal.

```js copy
console.log(5 == 5) // true
console.log(5 == '5') // true
console.log(5 == 8) // false
```

<Callout type="error" emoji="️🚫">
  Never use this equal operator. It makes your code more prone to error. Use the
  <strong>strict equal (===)</strong>
  instead.
</Callout>

### Not equal (!=)

```js copy
console.log(5 != 5) // false
console.log(5 != '5') // false
console.log(5 != 8) // true
```

<Callout type="error" emoji="️🚫">
  Never use this not equal operator. It makes your code more prone to error. Use the
  <strong>strict not equal (!==)</strong>
  instead.
</Callout>

### Strict equal (===)

```js copy
console.log(5 === 5) // true
console.log(5 === '5') // false
console.log(5 === 8) // false
```

### Strict not equal (!==)

```js copy
console.log('a' !== 'a') // false
console.log(5 !== '5') // true
console.log(null !== undefined) // true
```

### Greater than (\>)

```js copy
console.log(2 > 9) // false
console.log(9 > 2) // true
```

### Greater than or equal (\>=)

```js copy
console.log(2 >= 2) // true
console.log(9 >= 2) // true
console.log(1 >= 2) // false
```

### Less than (\<)

```js copy
console.log(5 < 9) // true
console.log(5 < 2) // false
```

### Less than or equal (\<=)

```js copy
console.log(4 <= 5) // true
console.log(9 <= 1) // false
console.log(6 <= 6) // true
```

## Logical operators

### Logical AND (&&)

| x     | y     | x&&y  |
| ----- | ----- | ----- |
| false | false | false |
| false | true  | false |
| true  | false | false |
| true  | true  | true  |

```js copy
console.log(true && false) // false
console.log(0 && 1) // 0
console.log('a' && 'a') // 'a'
console.log(true && true) // true
console.log(true && null) // null
```

### Logical OR (||)

| x     | y     | x\|\|y |
| ----- | ----- | ------ |
| false | false | false  |
| false | true  | true   |
| true  | false | true   |
| true  | true  | true   |

```js copy
console.log(true || false) // true
console.log(0 || 1) // 1
console.log('a' || 'b') // 'a'
console.log(true || true) // true
console.log(true || null) // true
```

### Logical NOT (!)

| x     | !x    |
| ----- | ----- |
| false | true  |
| true  | false |

```js copy
console.log(!true) // false
console.log(!false) // true
console.log(!'a') // false
console.log(!null) // true
console.log(!undefined) // true
console.log(!0) // true
console.log(!1) // false
console.log(!2) // false
```

## String operators

## Conditional (ternary) operator

```js
condition ? val1 : val2
```

```js copy
console.log(2 < 9 ? 'TRUE' : 'FALSE') // 'TRUE'
console.log(2 > 9 ? 'TRUE' : 'FALSE') // 'FALSE'

2 > 5
  ? console.log('2 is greater than 5, what?!')
  : console.log('2 is obviously, not greater than 5')
// '2 is obviously, not greater than 5'

let eatsMeat = true
const veredict = eatsMeat ? 'carnivore' : 'vegetarian'

console.log(`You are ${veredict}.`) // 'You are carnivore.'

eatsMeat = false
console.log(`You are ${eatsMeat ? 'carnivore' : 'vegetarian'}.`)
// 'You are vegetarian.'

const numbers = [1, 33, 23, 3]

const sizes = numbers.map((number) => (number >= 10 ? 'BIG' : 'small'))

console.log(sizes) // ['small', 'BIG', 'BIG', 'small']
```

Reads if condition is true, return val1, otherwise return val2

## Unary operators

### delete

The `delete{:js}` operator deletes an object's property.

```js copy
const item = {
  color: 'black',
  weight: 2,
  owner: null
}

console.log(item) // { color: 'black', weight: 2, owner: null }
delete item['owner']
console.log(item) // { color: 'black', weight: 2 }
```

### typeof

The `typeof{:js}` operator returns a string indicating the type of the operand's value.

```js copy
console.log(typeof 99) // 'number'
```

## Relational operators

### instanceof

The `instanceof{:js}` operator tests to see if the prototype property of a constructor appears in the prototype chain of an object.

```js copy
const philosopher = new String('Plato')

console.log(philosopher instanceof String) // true
```
